# Proposal v0.107 — real-time track slice 4 (the closure): broadcast + the §20 chat-room end-to-end

- **Status:** Draft, for approval.
- **Scope:** the track's **completion proof** — the §20 chat-room running end-to-end with multi-connection **fan-out**, plus the one primitive that makes a fan-out production-correct (`parTraverse`). This is the slice that *retires* the real-time / WebSocket track.
- **Provenance:** the broadcast / held-aware-iteration piece deferred since slice 2 ("the held-aware iteration borrow surface — `forEach`/`parTraverse` over connections") and named as the slice-4 closure by ADRs 0130–0134. Realises the §20 worked example in full (`bynk-design-notes.md` §20: `connections.values.parTraverse(c => c.send(…))`). **Security-bearing only by association** (it sends on already-authenticated held connections; no new boundary) — runs `/security-review` + `/code-review` to honour the track's posture, but expects a light review. ADR provisionally **0135**, version **0.107.0**.

## 1. The surprise — broadcast already works

A held-aware iteration broadcast over a `store Map[K, Connection]` **already compiles
and emits correctly on both targets today**, assembled incrementally:

```
on call broadcast(msg: ServerFrame) -> Effect[()] {
  let _ <- conns.forEach((c: Connection[ServerFrame]) => c.send(msg))
  ()
}
```

- **Bundle:** `for (const __x of Object.values(__state.conns)) { await (c => c.send(msg))(__x); }` — iterate the stored `TestConnection`s, send to each.
- **Workers:** the same, but the iteration resolves each stored **connId** to its live socket and skips closed ones: `Object.values(__state.conns).flatMap(cid => { const c = resolveConnection(this.state, cid); return c.tag === "Some" ? [c.value] : []; })`.
- The closure parameter `c` is a **borrowed** held binding (slice 2): `c.send(…)` is allowed; `c.close()`/transfer is `bynk.held.consume_on_borrow`; no disposal obligation.

So the **held-aware iteration borrow surface is built**. Slice 4 is therefore not a
large new surface — it is the **proof** that the §20 chat-room runs, plus the few
gaps that proof exposes.

## 2. Scope and non-goals

**In scope.**

- **`parTraverse` — the parallel broadcast primitive.** `forEach` awaits each send
  in turn, so one slow or half-dead client **head-of-line-blocks** the whole
  broadcast. `parTraverse` issues them concurrently (`Promise.all`), the §20 form and
  the production-correct one. A small sibling of `forEach` over a `Query`/`List`,
  with the same borrowed-closure-param semantics.
- **Exclude-self by key (DECISION B).** The §20 `connections.values.filter(c => c !=
  conn)` compares connections — but `Connection` is **non-comparable** by design (ADR
  0130, identity not value-equality). The slice expresses "everyone but the sender"
  by **key** instead (the sender's `UserId` is in scope), keeping connections
  non-comparable.
- **The §20 chat-room end-to-end (the completion proof):** multiple `TestConnection`s
  join a room; a `post` **broadcasts** the message to all; `join`/`leave` broadcast
  presence; a behaviour test under node asserts every connection received the
  expected frames (fan-out actually reaches all of them). On the bundle target, with
  no Durable Object.
- **The `.values` accessor (DECISION C, small):** support `<held-map>.values` as the
  explicit iteration form (today the bare `<held-map>` already lowers to the query;
  `.values` currently mis-parses as a cross-context path). Either wire `.values` or
  settle on bare-map iteration as the one surface and diagnose `.values` clearly.

**Non-goals (and why).**

- **Lambda parameter-type inference.** The §20 writes `(c) => c.send(…)` unannotated;
  today that warns (`bynk.lambda.unannotated_param`) and the slice annotates
  (`(c: Connection[ServerFrame]) =>`). Inferring `c` from the collection element type
  is a general ergonomic improvement **orthogonal** to the track — a named follow-on,
  not gating the closure.
- **The full §20 feature breadth as new work** — `Idempotency`, `Events`, `Log`
  history, `@retain` are **existing** capabilities; the proof *uses* them, it does not
  build them.
- **Backpressure / partial-failure delivery semantics of a broadcast** — a `send`
  that fails on one socket is the per-connection lifecycle's concern (the connection
  closes; the next iteration resolves it to `None`). A richer delivery contract is out
  of scope.

## 3. Decisions

**[D1 — add `parTraverse` as the parallel broadcast primitive].** `<collection>.parTraverse(f)`
where `f: (T) -> Effect[()]`, lowering to `await Promise.all(xs.map(x => f(x)))` —
the concurrent sibling of the existing sequential `forEach`. The closure param keeps
the borrowed-held semantics. *Recommended;* a sequential broadcast is a real
availability footgun (one stuck client stalls everyone), and it is the §20 form.

**[D2 — exclude-self by key, not connection identity].** "Broadcast to everyone but
the sender" filters on the **key** (`UserId`), not on `c != conn`. `Connection`
stays non-comparable (ADR 0130 unchanged). *Recommended;* it avoids reopening a
settled decision and reads as clearly (the sender's identity is already in scope).

**[D3 — the proof runs on bundle against `TestConnection`].** The end-to-end chat-room
behaviour test drives `open`/`message`/`close` and the agent's `post`/`join`/`leave`
with several `TestConnection`s and asserts the captured frames on each — the same
mechanism slices 3a/3b-iii used, now with **fan-out**. No real Workers runtime proof
(needs Miniflare/workerd); the Workers emission stays covered by `tsc --strict` + the
shape fixtures + the strip-types guard. *Recommended;* it is the only end-to-end proof
the unit harness can run, and `parTraverse`/broadcast lowering is target-shared.

**[D4 — the `.values` accessor].** Support `<held-map>.values` (and `<map>.values`)
as the explicit "iterate the values" form, sugaring to the same query the bare map
already yields. *Recommended;* it is what the design notes write and removes the
surprising "looks like a cross-context call" diagnostic. (If wiring it proves to pull
in disproportionate parser/resolver work, the fallback is to document bare-map
iteration as the surface and give `.values` a precise diagnostic — a call made during
implementation.)

## 4. Internal architecture

- **`bynk-check`:** `parTraverse` joins `forEach` in the query/list method kernels with
  the same effectful-closure typing and the borrowed-closure-param linearity (the
  existing `forEach` borrow path generalises). The `.values` accessor resolves on a
  `Map`/held-`Map` receiver to the values query.
- **`bynk-emit`:** `parTraverse` lowers to `Promise.all(xs.map(f))`; `forEach` stays
  the sequential `for … await`. The held-`Map` resolution scan (3b-ii) is reused
  unchanged — `parTraverse` over a held map resolves connIds exactly as `forEach`
  does. `.values` reuses the bare-map query lowering.
- **No runtime changes** beyond what exists (`resolveConnection` etc.).

## 5. Risks

- **`parTraverse` is the only genuinely new language surface** — kept minimal (one
  method, mirroring `forEach`). If it expands (ordering guarantees, concurrency
  limits), that is scope creep to resist; v1 is unordered `Promise.all`.
- **The `.values` accessor may be more parser/resolver work than it looks** (it
  currently mis-resolves as a context path) — D4's fallback bounds that risk.
- **No real-broadcast runtime proof on Workers** — stated honestly; the bundle
  behaviour test proves the fan-out logic, Workers stays at `tsc --strict` + shape
  snapshots. Never claim a `wrangler dev` proof that did not run.
- **Track-retirement scope discipline** — "the §20 chat-room end-to-end" must mean the
  *worked example*, not an open-ended feature pile; the proof asserts fan-out +
  presence, not a production chat product.

## 6. Testing

- A bundle behaviour test: N `TestConnection`s join one room; `post` broadcasts; assert
  every connection captured the broadcast frame; `join`/`leave` presence reaches the
  others; the sender is excluded where §20 excludes it.
- Workers shape fixture: `parTraverse`/broadcast over the held connId map emits +
  `tsc --strict`-clean (resolution scan + `Promise.all`).
- Negatives: `parTraverse((c) => c.close())` → `bynk.held.consume_on_borrow` (the borrow
  holds under the parallel primitive too); exclude-self by `c != conn` → the
  non-comparable diagnostic.
- Full gate `BYNK_REQUIRE_TSC=1 cargo test --workspace --locked` + `fmt` + `clippy`.

## 7. Rollout — the track retires

ADR 0135; version 0.107.0; changelog + the §2.9 / §20 spec status moved to **complete**;
the real-time / WebSocket **track doc** marked retired (all slices shipped); delete
this proposal on merge. `/security-review` + `/code-review` (light). With slice 4 the
track that began with `Stream[T]` (slice 0) closes: the §20 chat-room — edge auth, a
held connection transferred to an agent, surviving hibernation, inbound frames
decoded and dispatched, and a message fanned out to every connection in the room —
compiles and runs end-to-end.
