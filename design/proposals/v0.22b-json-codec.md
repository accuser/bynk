# v0.22b — Typed JSON codec + boundary-`Int` tightening

- **Status:** Draft (proposal). **Merging is approval to build.**
- **Builds on (verified):** **`Float` (v0.21, #29, landed)** — decimal JSON, and 0040's
  finite-boundary rule is the template for the bare-`Int` tightening; **the boundary codec
  machinery** — `serialise_<T>` / `deserialise_<T>` (`emitter/serialisation.rs`) **and the
  workers handler-param path** (`emitter/workers_entry.rs`, `deserialise_call` at `:544`);
  **the `ValidationError` precedent** (`ast.rs:973`, `checker.rs:46`) — a compiler-known,
  Karn-visible boundary error, which `JsonError` follows; **v0.20b type-application**
  (`name[T]`, the 0039 neighbourhood). *Independent of v0.22a* — orderable either way; the
  numbering assumes a-then-b.
- **Spec sections touched:** §4 (type-app on qualified statics), §5 (static semantics), §6
  (type system — `JsonError`), §7 (emission — incl. the bare-`Int` codec change), §9
  (diagnostics), Appendix B.
- **Scope:** the typed `Json` codec, a compiler-known `JsonError`, type-application on
  qualified statics, and the **bare-`Int` integrality tightening** (the re-bless). No new
  value-method surface — that is v0.22a.

## Settled (recorded as ADRs — this slice: 0045, 0047, 0049)

- **▸ Typed JSON codec only.** `Json.encode(v) -> String` and `Json.decode[T](s) ->
  Result[T, JsonError]`, **compiler-backed**: `encode` dispatches to `serialise_<typeof v>`
  + `JSON.stringify`; `decode[T]` to `JSON.parse` + `deserialise_<T>`. Must be compiler-known
  (an erased generic can't conjure the per-type `deserialise_<T>`). Untyped `Json` is out of
  scope.
- **▸ `Json.encode` is *not* total** *(0040).* `encode` wraps `serialise_`, and 0040 makes
  serialising a non-finite `Float` **throw** (a contract violation). So `encode(v) -> String`
  but it **throws on any `v` containing a `NaN`/`Infinity`** `Float`. Keep the `-> String`
  signature with a **normative contract-violation sentence** — `Result`-ifying `encode` for a
  state the program itself created would be noise, and it matches the boundary posture.
- **▸ `JsonError` is a compiler-known built-in** *(ValidationError precedent).* Today
  `BoundaryError` is TS-runtime-only — Karn programs never see it (workers turn it into a
  400). `Json.decode` returning `Result[T, JsonError]` puts a boundary failure **in the
  program's hands**, so `JsonError` must be a compiler-known type the checker and runtime
  both know — exactly like `ValidationError` — **not** a Karn-declared sum in a commons. It
  carries a `Malformed` case (for `JSON.parse` failure) alongside the structural-mismatch
  case, and **surfaces the field path** the deserialiser already tracks.

## Decisions (open — settle at review)

**[DECISION] Bare-`Int` boundary fields must validate integrality (wire-contract
tightening).** Today a bare `Int` field at the boundary checks `typeof === "number"` **only**
— no `Number.isInteger`. This is lax in **two** places, and v0.21's `Number.isFinite`
precedent patched **both**, so both must be tightened together (or record fields tighten
while HTTP params stay lax):
- record/nested fields — `emitter/serialisation.rs` (the `BaseType::Int => "number"` arm;
  `isInteger` lives only in refined `.of`, `emitter.rs:1566`);
- **workers handler params** — `emitter/workers_entry.rs` `deserialise_call` (`:544`), the
  `typeof`-only arm at `:564` (where the `Float` `isFinite` check sits at `:560`).

So `deserialise_Order({qty: 1.5})` *succeeds today*, and `Json.decode[Order]` would produce
an "`Int`" holding `1.5`. *Rec:* **add `Number.isInteger` to bare-`Int` at both sites**
(mirroring 0040's `Number.isFinite` for `Float`). Consequences — why this is a *settled
decision, not a fixture line*:
- a **behaviour change to every emitted codec and handler** with a bare-`Int` field;
- an **explicit re-bless of prior positive fixtures** — a deliberate **exception to the
  byte-stability guard**;
- a **tightening of the wire contract** (a previously-accepted `{qty: 1.5}` now `Err`s).
*Rationale:* with `Float` in the language there is no excuse for fractional `Int`s; codec and
`.of` should agree (0040 aligned `Float`). *Alternative:* leave bare `Int` lax — rejected; a
latent correctness hole now visible.

**[DECISION] `Json.decode[T]` — type application on a qualified static.** `ConstructorCall`
(`ast.rs:1048`) has **no `type_args`** (unlike `Call`, `ast.rs:1019`), so `decode[T]` is an
**AST + parser + checker** change, and the v0.20b same-line-`[` rule must be restated for
`decode[`. This is the hardest item in either slice, and **independent of v0.22a**.
*Sub-decision — the domain of `T`:* named types only, or **any boundary-legal type-ref**
(`Json.decode[List[Order]]`)? *Rec:* **any boundary-legal type-ref** — the per-instantiation
helper machinery exists from v0.20b — though it changes the §5 prose and the fixture set.
Without type-app, `decode` can only infer `T` from an expected type, and
`karn.types.uninferable_type_arg` fires when neither is present.

**[DECISION] `JsonError` shape.** *Rec:* a compiler-known type (the `ValidationError`
machinery) carrying a `Malformed` case (`JSON.parse` failure) + the structural-mismatch case,
reusing `BoundaryError`'s runtime shape and the tracked field path.

JSON decode failures — malformed, shape-mismatch, **and now non-integer-into-`Int`** — are
runtime `JsonError` values, **not** diagnostics.

## Fixtures (shape — not normative emission)

- **Positive** — **JSON round-trip** for a record with a **`Float` field**
  (`{"price": 19.99}`, proving the v0.21 dependency); `decode[T]` incl. `decode[List[Order]]`
  if the domain extends. Each passes `tsc --strict`.
- **Boundary tightening** — a **non-integer JSON number into a bare-`Int` field now `Err`s**,
  at **both** a record field and a handler param *(a positive assertion of the tightened
  codec — replacing the original draft's negative fixture that asserted behaviour which did
  not exist)*.
- **Negative** — `Json.decode[Order]("{")` → `Err(Malformed)`; a shape mismatch (missing
  field / wrong type) → `Err`; `Json.decode(s)` with no target type → `uninferable_type_arg`.
- **Re-bless (explicit exception to byte-stability)** — adding `Number.isInteger` changes
  every emitted codec **and handler** with a bare-`Int` field; those prior positive fixtures
  are **re-blessed**, diff reviewed. All *other* prior fixtures stay byte-stable.

## Risks
- **Bare-`Int` re-bless across both sites** — a mass re-bless is where regressions hide; the
  byte-stability exception must be deliberate and the diff reviewed, and **both**
  `serialise`/`deserialise` and the workers handler path must move together.
- **Type-app on qualified statics** — `ConstructorCall` AST + parser + checker change; the
  first thing to build, gated by the domain-of-`T` sub-decision.
- **`JsonError` alignment** — compiler-known (not a commons sum); align checker, runtime, and
  the `ValidationError`-style machinery; surface the tracked path.
