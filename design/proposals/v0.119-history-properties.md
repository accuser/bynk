# v0.119 — history properties: `property` over a generated call-history (testing track, slice 7)

- **Scope:** a single-increment **language + test-runtime** change — the eighth and final
  testing-track slice, and the top rung of the spine. It adds one surface and no removals:
  - a **`History[Agent]` generator** in `for all` position inside a `property` — the runner
    generates a bounded, seeded, random **sequence of the agent's handler calls**, drives the
    **real** handlers from the agent's initial state, and binds the observed **history** (the
    ordered list of driven steps, each carrying its call, its outcome, and the state it
    reached) for the predicate to judge.

  It adds **no new logic**: a history is a `List[Step]` asserted with the `List` surface a
  reader already knows (`.all` / `.any`, indexing, `.length()`), exactly as slice 5's
  `trace(Cap.op)` is. "Every spend has a prior top-up" is a list quantifier over a bounded
  generated run — not a temporal operator, not a matcher. Version → **v0.119**.
- **Realises:** the spine's sixth rung — `value → domain → call → snapshot → step → history`
  (`design/tracks/testing.md`, Pillar I) — and the **sound route for agent behavioural
  generation** the track commits to under DECISION P: an agent's behaviour is generated by
  driving **handler sequences** and observing the states they *reach*, never by fabricating a
  valid-but-unreachable state. It closes the invariant↔test duality for agents: `invariant`
  judges a reached snapshot and `transition` a reached step at the commit boundary; a history
  property judges a whole reached *run*, adversarially, in the runner.
- **Relates:** ADR 0144 (one predicate surface — a history predicate is the ordinary
  predicate/`List` surface, so there is no LTL vocabulary and no history matcher library);
  ADR 0149 (generation is valid inhabitants — a step's generated arguments are drawn from each
  handler parameter's refinement domain, reusing the slice-2 generator); ADR 0151 (the
  invariant subject widens to the step — a history *is* the ordered sequence of those steps,
  now the generated subject of a runner-only property); ADR 0152 (observation records at the
  capability seam — a driven handler's capability calls are still recordable, so `trace(Cap.op)`
  composes inside a history predicate); ADR 0153 (tier is a dial — a `property` is `case`-only
  for tiers, so a history property carries **no** `as`: it is in-process and generative by
  construction, the flake-free tier).
- **ADR at implementation:** one, load-bearing, numbered at authoring (next free after 0154;
  do not pre-allocate a range — `packaging.md` is also in flight):
  - **H** — *history/protocol properties are test-runner-only, over a generated call-history
    driven through the real handlers; there is no commit-time temporal enforcement, and the
    temporal reach is bounded `List` quantification over a generated run, not a new logic.*
    Cites ADR 0144 (no new predicate surface), ADR 0149 / DECISION P (reachability by
    driving, not fabrication), and ADR 0151 (the step is the unit a history sequences).

## The surface

A **history property** is a `property` whose generated subject is a *run* of an agent:

```bynk
suite Wallet {
  property "no spend without a prior top-up" {
    for all run: History[Wallet] {
      expect run.all((s) =>
        s.call is Spend implies run.upTo(s).any((p) => p.call is TopUp and p.accepted))
    }
  }
}
```

`for all run: History[Wallet]` binds `run` to a **generated, driven** history of the `Wallet`
agent. The runner:

1. **generates** a bounded random sequence of `Wallet`'s handler calls — each handler chosen
   uniformly, each argument drawn from its parameter's refinement domain (the slice-2
   generator, so the same seed and shrinker apply);
2. **drives** the sequence against a **fresh** `Wallet` instance from its initial state,
   invoking the **real** handlers and their real invariants (so every state in `run` is one a
   handler actually reached — reachability by construction, DECISION P);
3. **binds** `run` to the observed history and evaluates the predicate; on failure it reports
   the seed and a **shrunk** minimal failing sequence.

### A history is a `List[Step]`

`run` is an ordinary `List`, asserted with the surface you already know — there is no
history-only iteration construct, mirroring slice 5's `trace(...)`. Each `Step` record carries:

| Field | Type | Meaning |
|---|---|---|
| `.call` | a generated sum over the agent's handlers (`Spend { amount }`, `TopUp { amount }`) | which handler ran, with its generated arguments, matched with `is` / `match` |
| `.accepted` | `Bool` | whether the handler **committed** a new state (vs. rejected — an invariant/`requires` refusal, `Err`) |
| `.old` / `.new` | the agent's state type | the committed old→new pair (the same `old`/`new` a `transition` sees), so a step is a reached edge of the state graph |

The typed step shape resolves the track's open question — *"the typed shape of `old` / `new` /
a bound `trace(...)` such that `is` and `implies` apply unchanged and `impure_predicate` still
bites."* A step's fields are all pure values; a history predicate is a pure `Bool` over them, held to
the same predicate-purity discipline as every other rung (the `impure_predicate` family —
`bynk.invariant` / `bynk.transition` / `bynk.contract` — and `bynk.expect.not_bool`), so `is`
and `implies` read over `.call` / `.old` / `.new` unchanged.

The one addition the ordered surface needs is a **prefix accessor**, `run.upTo(step)` (the
history strictly before `step`) — see DECISION C for whether it is a `History` accessor or
folds into the general `List` surface.

## Decisions

**[DECISION H — the gate] History properties are test-runner-only; no commit-time temporal
enforcement.** Cross-step "eventually / always" claims are evaluated **only** by the runner,
over a **bounded, generated** call-history — never at the commit boundary. Commit-time stays
where slices 4–5 put it: `invariant` over a reached snapshot and `transition` over a reached
step. *Recommend adopt* (the track's standing recommendation, flagged for sign-off): it keeps
the runtime cheap, keeps the flake-free guarantee scoped to the lower tiers, and — crucially —
keeps a *bounded* history out of the LTL / model-checking surface an unbounded commit-time
temporal logic would drag in. **This is the load-bearing ADR; the whole slice sits behind it.**

**[DECISION A] The temporal reach is `List` quantification, not new operators.** "Always P"
is `run.all((s) => P)`; "P before Q" is a quantified prefix check via `run.upTo(...)`;
"eventually P" is `run.any((s) => P)`. *Recommend* adding **no** `always` / `eventually` /
`before` keywords and **no** history matcher library — the reach is what `List` + the predicate
surface already give over a finite run (ADR 0144). Consequence, stated honestly: **unbounded**
"eventually" (liveness over an infinite run) is *not expressible*, by design — a bounded run
can only witness bounded reach. This is the deliberate ceiling DECISION H buys; the docs must
say so plainly so no reader mistakes it for model-checking.

**[DECISION B] `History[T]` requires `T` to be an *agent*, and is generative-only.** Only an
agent has handlers to sequence and reachable states to observe; a value or a service does not.
`History[Value]` is `bynk.history.not_an_agent`. `History[...]` is legal **only** in `for all`
position inside a `property` (it *is* a generator): elsewhere it is `bynk.history.outside_property`.
Because a `property` is `case`-only for tiers (ADR 0153), a history property carries no `as`
and needs no new tier rule — it runs in-process against the real handlers, on the generative,
flake-free tier, and `bynk.tier.property_has_tier` already forbids an `as` on its header. The
agent must be **drivable**: every handler parameter must be refinement-generable (the `for all`
rule), else `bynk.history.not_generable`.

**[DECISION C — open] `run.upTo(step)` — a `History` accessor, or a general `List` slice?**
The ordered predicate needs "the history before this step." Two forms: **(a)** a small
`History`-typed accessor `run.upTo(s)` / `run.after(s)` that reads at the call site; **(b)** a
general `List.takeWhile` / index-slice that any `List` gets, with the testing layer adding
nothing. *Recommend (a) for now* — it reads at the point of use and keeps the slice
self-contained — but flag that if the general `List` surface grows slicing independently, the
accessor should collapse into it rather than persist as a test-only method (the slice-5
precedent: "there is no test-only iteration construct"). Open, to settle at implementation.

**[DECISION D] A history property that merely restates a snapshot/step invariant is
redundant.** `for all run: History[Wallet] { expect run.all((s) => s.new.balance >= 0) }` when
`Wallet` carries `invariant nonneg: balance >= 0` re-checks a guarantee every reached state
already has (the driver only commits states that satisfy the invariants). *Recommend* a
**conservative, syntactic** flag `bynk.history.restates_invariant` — α-equivalent predicate
over `.new` (or `.old`/`.new`) versus a declared `invariant` / `transition` — mirroring the
existing `bynk.property.restates_refinement`. Never full semantic equivalence (a false
rejection blocks a valid test); near-duplicates slip through by design.

**[DECISION E] Bounding & shrinking is seeded, length-bounded, and shrinks the *sequence*.**
Reuse the slice-2 property infrastructure: a fixed root seed by default, a bounded run length
(a `bynk test` knob, small default) and case count, boundary-biased argument sampling, and a
printed repro command. On failure the shrinker **delta-debugs the step list** (drop steps,
**re-driving** after each reduction so the reduced sequence stays *reachable*) and then shrinks
the surviving arguments with the value shrinker. *Recommend adopt.* The deeper shrink strategy
(how aggressively to minimise, how to bound re-drives) is an implementation-tuning risk, not a
language decision — see Risks.

**[DECISION F — scope boundary] Single-agent histories now; multi-agent / cross-context
protocols are a named follow-on.** A `History[Agent]` sequences **one** agent's handlers. A
protocol *across* agents or across the `system`-tier wire (order in context A eventually
settles a ledger in context B) is the LTL-flavoured surface DECISION H warns about; it is
**out of scope** for v1 and named here so the boundary is deliberate. This answers the track's
open question — the history rung earns its place in v1 as *single-agent* behavioural
generation; the *multi-agent protocol* tail ships later (or never) without blocking the rest.

**[DECISION G] Version — minor v0.119; language + test-runtime.** New generative surface with
fixtures; no removals, so no migration/codemod. The book's currency banner advances via
`scripts/bump-version.sh 0.119.0`.

## Risks & mitigations

- **State-space explosion / cost.** An agent with many handlers and wide argument domains has
  an enormous run space. *Mitigation:* bounded run length + case count (DECISION E), a small
  default, boundary-biased sampling; the property surface's existing cost discipline applies.
- **Shrinking to a *reachable* minimal counterexample.** Dropping a step can make the suffix
  undrivable (a `spend` after the `topup` that funded it). *Mitigation:* the shrinker re-drives
  after every reduction and keeps only reductions that still reproduce the failure — slower, but
  the counterexample it prints is guaranteed reproducible. Bounding the re-drive budget is the
  tunable risk.
- **Reproducibility depends on real capabilities being injected.** A driven handler that reads
  ambient nondeterminism through an *un-provided* capability would make the run irreproducible.
  *Mitigation:* it can't — subjects have exactly three honest sources (Pillar II); the
  generative tier provides every capability seam deterministically (real or `provides`-stubbed,
  slice 6 composes), so the whole run is a function of the seed.
- **The bounded-reach ceiling reads as a limitation.** Users from a model-checking background
  may expect unbounded liveness. *Mitigation:* DECISION A's ceiling is documented as a
  deliberate design choice, with the `system`-tier / policy guarantee (DECISION U, still open)
  named as where an *always-on-every-path* universal would live instead — a history property is
  a runner sample over generated runs, never a proof.
- **`restates_invariant` equivalence detection.** Same hazard as the slice-2/3 redundancy
  checks. *Mitigation:* conservative syntactic α-equivalence only (DECISION D); a false
  rejection is worse than a miss.
