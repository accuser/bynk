# v0.27 — LSP inlay hints (inferred types)

- **Status:** Draft (proposal). **Merging is approval to build.** *(The A-2 headline — making
  Karn's accumulated *inference* legible. Touches `karnc` (a hint sink in the checker) + `karn-lsp`.)*
- **Builds on (verified):**
  - The checker computes types and **`Ty::display()`** (`checker.rs:87`) renders a label — so the
    label data exists. But the typed model is **not** harvestable as-is (below).
  - **The typed model is dropped on any error.** `check_record` returns `Ok(TypedCommons { …,
    expr_types })` **only `if errors.is_empty()`**, else `Err` (`checker.rs:179-190`); the project
    loop then `continue`s (`project.rs:1832+`). So a model carried in the `Ok` payload **vanishes
    file-wide on a single transient error** — exactly while the user is typing. The fix is
    **sink-style** retention, the v0.25 `RefSink` precedent (`refs` is a `&mut` sink that survives
    the error-`continue`).
  - **Raw `expr_types` can't position a hint.** A `HashMap<Span, Ty>` can't answer "which `let`
    values are unannotated, and where is the binding *name*?" — that needs the AST. The test
    contract already implies the right shape: the datum is keyed by the **binding-name span**
    (`name.span`), which `expr_types` never contains. So **harvest a curated per-file hint set**,
    not the raw map.
  - Hintable sites carry **optional** annotations (hint only when absent): `LetStmt.type_annot`
    (`ast.rs:933`), `LambdaParam.type_ref` (`ast.rs:820`); the checker already special-cases `_`
    bindings (`checker.rs:1429/1508`) and binds each lambda-param type into scope **by name**
    (`checker.rs:2587`) — `p.name.span` is where the *new* hint recording goes, not existing
    plumbing. **`let x <- e` (`EffectLet`)** binds the **peeled `Effect[T]` payload**
    (`checker.rs:1450`), *not* the rhs type.
  - `analyse_project`/`ProjectAnalysis` retains `index` + `errors` + `snapshots` only
    (`project.rs:341-349`); the LSP advertises no `inlayHintProvider`. Positions convert against the
    cached snapshot (v0.24); the v0.25 pure-function harness applies.
- **Spec sections touched:** `design/karn-lsp-spec.md` (Inlay hints + the capability; §4.3);
  `karnc` (a hint sink threaded through the checker; `ProjectAnalysis` retains the harvested set).
- **Scope:** **inferred-type** inlay hints for `let` / `let <-` bindings and lambda parameters.
  **Not** here: generic-instantiation hints (deferred), parameter-*name* hints, `inlayHint/resolve`
  tooltips, semantic tokens.

## Settled (recorded as ADRs ≥0056)

- **▸ Hints are a curated set harvested from the checker via a sink — not raw `expr_types`, not a
  tool-side re-inference.** A **hint sink** (the `RefSink` analogue) records `(binding-name span,
  label)` at each hintable site as the checker computes the binding's type. `ProjectAnalysis`
  retains the per-file `Vec<(Span, String)>` — small, AST-positioned, and **no `Ty` leaks through
  the public surface**. Same extend-`Analysis` pattern as v0.25's index and v0.26's diagnostics.
- **▸ The sink survives errors (for the sites the checker still reaches); positions via the cached
  snapshot.** Because the sink is a `&mut` parameter (not the `Ok(TypedCommons)` payload
  `check_record` drops), hints **persist through a transient error** rather than flickering
  file-wide. *Caveat:* a `check_record` `Err` (any fn-body type error) `continue`s **before**
  `check_v0_5_declarations` runs (`project.rs:1832-1838`), so that file's **handler-body** hints are
  suppressed until the error clears — other fn-body hints survive. The guarantee is "sites the
  checker still reaches," not file-total. Hints appear only where the annotation is absent; no cached
  round (pre-first-analysis / non-project file / a group whose composition failed) → no hints.

## Decisions (open — settle at review)

**[DECISION] A hint sink threaded through the checker (not retained `expr_types`).** *Rec:* add a
`HintSink` (mirroring `RefSink`, **including its `enter_file` per-file attribution** — spans are
bare byte offsets) passed `&mut` through `check_record` **and** `check_v0_5_declarations`
(`project.rs:1861` — handler bodies are typed there, *after* `check_record`, so handler `let`s only
get hints if the sink reaches it). At each hintable binding the checker records `(name.span,
": " + final_ty.display())`. `ProjectAnalysis` gains a per-file `Vec<(Span, String)>`; the LSP's
`Analysis` retains it; `textDocument/inlayHint` (carrying a **visible range**) returns the entries
in range, positioned via the snapshot.

**[DECISION] Record the binding's *final* type at the site — `=`, `<-`, and lambda params.** *Rec:*
- **`let x = e`** (`type_annot == None`, `name != "_"`): record the rhs binding type.
- **`let x <- e`** (`EffectLet`): record the **peeled `Effect[T]` payload** (`checker.rs:1450`) — the
  binding's actual type — *not* `Effect[T]`. Harvesting `final_ty` at the arm gets both `=` and `<-`
  right (and is more honest than `expr_types[e.span]`, which only coincides with `=` today).
- **Lambda params** (`type_ref == None`): record at `p.name.span` where the checker already has the
  inferred `ty` (`checker.rs:2587`) — a recording call, *not* a widening of `expr_types`.
- **Skip** `_` bindings (already special-cased) and **synthetic / `test` / `integration` units**
  (the `assemble_index` precedent, `project.rs:436`).

**[DECISION] Deferred — generic instantiations.** `name[?](…)` → the inferred `[T]`: the
monomorphised type arguments are **not stored queryably**; this returns once the checker records the
inferred instantiation (the high-value-but-deferred one, like v0.26's InRange swap).

**[DECISION] Label fidelity — pin the `Fn` rendering.** `Ty::display()` renders `Unit` as `()` and
`Fn` as `A -> B` with a parenthesisation rule (`checker.rs:100-116`). The display-fidelity fixture
**must include a lambda-typed binding** (the variant most likely to read as non-surface), plus
`List`/`Option`/refined cases. Hint shape: `: T`, padding-left, `InlayHintKind.Type`.

**[DECISION] Visibility & toggles.** *Rec:* the server always *produces* hints; the **client
controls visibility** (VS Code's built-in inlay toggle + a `karn.inlayHints.enable` / types-only
setting — a B-1 surface item, no `karnc` change). `inlayHint/resolve` tooltips are out of scope.

## Tests (the LSP analogue of fixtures)

- **`karnc`** *(`cargo test`, no LSP):* a fixture project asserting hint data `(name.span, label)` for
  a `let =` inferred binding, a **`let <-` binding** (label is the *payload* type, not `Effect[…]`),
  and an unannotated lambda param; a **display-fidelity** case **including a lambda-typed binding**
  (pins the `Fn` rendering); an **annotated** `let`/param and a **`_`** binding assert **no** hint;
  and a file with **one fn-body type error** still yields hints for the **other fn bodies** (the
  sink-survives-errors guarantee, scoped to sites the checker reaches — *not* handler bodies, which
  the `check_record` `Err` short-circuits before the v0.5 pass).
- **`karn-lsp` pure-function level** *(the v0.25 harness — no transport):* a function computing inlay
  hints from the retained set + a range returns the expected `(position, label)`; out-of-range and
  no-cached-round return empty. The `inlayHintProvider` advert is a **trivial unit check**.

## Risks
- **Sink placement** — must thread through both `check_record` and `check_v0_5_declarations` so
  handler-body `let`s get hints, and survive the per-step error-`continue`s; the dominant care point.
  **Bounded guarantee:** a fn-body error `continue`s before the v0.5 pass, so that file's
  *handler-body* hints are suppressed until it clears — the survives-errors claim is scoped to "sites
  the checker still reaches," not file-total. *(Restructuring the `continue` to run v0.5 checks anyway
  is scope creep — out.)*
- **`EffectLet` payload** — record the peeled type, not `Effect[T]`; covered by a dedicated fixture.
- **Label fidelity** — `Fn`/`Unit` rendering pinned by the fixture.
- **Staleness** — positions via the cached snapshot (v0.24), as for code actions.

## Definition of done

A `HintSink` threads through `check_record` + `check_v0_5_declarations`, recording `let =` / `let <-`
(peeled payload) / lambda-param hints at the binding-name spans, skipping `_` and synthetic/test
units; `ProjectAnalysis` retains the per-file `Vec<(Span, String)>` and the LSP `Analysis` keeps it
(same shape as v0.25/v0.26); the LSP advertises `inlayHintProvider` and returns inferred-type hints
for a visible range, **served from the cached set, positioned via the snapshot, empty when no model
is cached**, and **hints persist through a transient type error at the sites the checker still reaches**
(fn-body errors short-circuit that file's handler-body hints before the v0.5 pass); `karnc` unit
tests cover the hint data (incl. `EffectLet`, a `Fn`-display-fidelity case, annotated-and-`_`-
suppress-hint, and survives-one-error scoped to other fn bodies) and the LSP computation is a **pure-function** test (no transport claimed);
`design/karn-lsp-spec.md` gains the Inlay-hints section and §4.3 is updated; decision record added;
changelog + LSP-spec delta land together; `cargo test`, `clippy -D warnings`, `fmt --check` green.
