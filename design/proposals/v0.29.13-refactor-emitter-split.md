# v0.29.13 ‚Äî Refactor track: split `emitter.rs` into named submodules

- **Status:** Draft (proposal). **Merging is approval to build.** *(Applies the named-concern-modules
  convention, ADR 0060, to the largest remaining file. Independent; the only prerequisite is the
  emitter golden/`tsc_verify` fixtures. Behaviour-preserving ‚Äî emitted TS is byte-identical.)*

## Context ‚Äî the largest file the v0.29 track left behind

- `emitter.rs` is **5,772 lines** ‚Äî the single largest source file. It already has an `emitter/`
  directory (`workers`/`workers_entry`/`serialisation`/`wrangler`), but **the parent is still 5.8k** ‚Äî
  the exact "a submodule dir does not mean the parent is small" trap ADR 0060 names.
- It is **free-function-dominated** (134 free `fn`, 8 `&self` methods, 2 structs), so ‚Äî exactly like
  the `project.rs` (v0.29.2) and `checker.rs` (v0.29.10) splits ‚Äî the proven flat-resolution recipe
  applies directly: move the free functions into named submodules behind a parent facade via
  `use super::*` + parent glob re-export.

## Builds on (verified)

- **The function names cluster cleanly** (verified counts): `lower_*` (30), `emit_*` (31), `ts_*` (7),
  `collect_*` (7), plus worker/cross/refined helpers. Two of those are large, coherent seams:
  - **`lower_*` (30 fns, ~`:3590`‚Äì`:5200`) ‚Äî the expression/statement lowering.** `lower_expr`,
    `lower_method_call`, the per-kernel set (`lower_{list,numeric,string,option,result,map}_kernel`),
    the per-arm set (`lower_call`/`lower_bin_op`/`lower_constructor_call`/`lower_ident`/`lower_if`/‚Ä¶),
    plus `lower_tail_expr`/`lower_mock`/`lower_json_codec_call`/`lower_cross_context_service_call`.
    v0.29.7 already organised these into clean per-arm/per-kernel helpers, so they are **ready to
    move** ‚Äî and they are ~half the file. **`emitter/lower.rs` is the single biggest clean win.**
  - **`emit_*` (31 fns) ‚Äî the emission functions.** `emit_commons_module`, `emit_method`,
    `emit_free_fn`, `emit_handler`, `emit_composition_root`, `emit_runtime_module`, `emit_tsconfig`, the
    worker-entry wrappers. ‚Üí `emitter/emit.rs` (optionally split by concern, see below).
- **The acceptance gate is already comprehensive.** Unlike `checker.rs` (sparse `#[test]`, needing the
  slice-0 pins), the emitter's output is pinned end-to-end by the golden fixtures **plus the
  `tsc_verify` strict-compile gate** ‚Äî so a byte-identical move is fully verifiable without new pins.
  (A handful of pure helpers ‚Äî `ts_type_ref`/`ts_ty`/`escape_ts_string` ‚Äî are already directly pinned
  from v0.29.1/v0.29.8.)

## Proposed seam map

| submodule | jobs moved | representative fns |
|---|---|---|
| `emitter/lower.rs` | expression/statement lowering | `lower_expr`, `lower_method_call`, `lower_*_kernel` (6), the per-arm `lower_*` (from v0.29.7), `lower_tail_expr`, `lower_mock`, `lower_json_codec_call`, `lower_cross_context_service_call` |
| `emitter/emit.rs` | module/decl/handler emission | `emit_commons_module`, `emit_method`, `emit_free_fn`, `emit_handler`, `emit_composition_root`, the runtime/tsconfig emitters |
| `emitter.rs` (parent) | the facade + shared core | `LowerCtx`, `ts_type_ref`/`ts_ty`/`ts_base`/`escape_ts_string`, `INDENT_STEP`, the `collect_*`/refined helpers, the public entry points, the existing `pub use` re-exports |

*(Exact assignment settles at implementation; the `collect_*` and cross-context helpers sit on a
boundary and go wherever the compiler's privacy rules make cleanest.)*

## Decisions (open ‚Äî settle at review)

**[DECISION] How finely to split `emit_*`.** *Rec:* start with **one `emitter/emit.rs`**; only split it
further (by concern ‚Äî `emit/modules.rs`, `emit/composition.rs`, ‚Ä¶) if it is itself still large after the
`lower_*` extraction. The `lower_*` removal alone takes the parent from 5.8k to ~2.7k; `emit.rs` need
not be perfect on the first pass.

**[DECISION] Slice into PRs.** *Rec:* **two PRs** ‚Äî `emitter/lower.rs` first (the biggest, cleanest,
self-contained seam), then `emitter/emit.rs`. Each golden-verified; the `CodeWriter`-style trap is
absent here (these are `-> String` lowerers, not indented writers ‚Äî see the shelved v0.29.6).

## Spec delta

**None.** No language surface, no `docs/src/spec/` chapter ‚Äî module reorganisation only.

## Tests

- **No new fixtures.** The emitter golden fixtures + `tsc_verify` are the gate; all pass **unedited**.
  The already-landed `ts_type_ref`/`escape_ts_string` pins move/stay with their subjects.
- Content-preservation check (every top-level item present exactly once across the new tree), as in
  v0.29.2/v0.29.10.

## Diagnostics / Risks

- **Lowest-risk of the remaining splits** ‚Äî free functions, flat resolution, comprehensive golden +
  `tsc_verify` coverage. The `lower_*` cluster is already helper-organised (v0.29.7).
- **`pub(crate)` cascade** ‚Äî compiler-driven, only where a cross-module call demands; the parent's
  core (`ts_type_ref`, `LowerCtx`) stays reachable to submodules via `use super::*`.
- **Reviewer fatigue** ‚Äî a large mechanical diff; mitigate with the two-PR split and the
  content-preservation gate (the diff is dominated by pure relocation).

## New decision records this increment adds

- **None.** Covered by ADR 0060 (named-concern-modules) and the track-level ADR 0059.
