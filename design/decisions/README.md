# Decision records

The design decisions that shaped Bynk's incremental evolution, one record per
language-defining call: `Status / Context / Decision / Consequences`, each
citing the [normative spec](../../site/src/content/docs/book/spec/index.md) section that now
embodies it. Records were harvested from the retired per-increment instalment
documents (see the spec's Appendix B); from v0.19 on, each increment adds its
records here as part of the increment itself.

A record is **immutable once accepted** ‚Äî a reversal is a new record that
supersedes the old one, the way 0016 supersedes the interim portability tiers.

The table below is **complete by construction**: a CI drift guard (`bynkc`'s
`decisions_index` test, run by the `drift` job) fails if an ADR file has no row
or a row links to no file. Summaries and statuses are curated by hand; the
*presence* of every record is enforced.

| [0184](0184-map-query-key-exposure.md) | **Map queries expose keys through `.entries`/`.keys`/`.values`; an entry is the nominal `MapEntry[K, V]` record, not a tuple** (v0.158, **closes #547** ¬ß8 P1 #8) ‚Äî a `Query` over a `store Map[K, V]` yielded values only, so every keyed collection duplicated its key inside the value (todo's `TodoItem.id`); the type-system sketch named `Map[K,V].entries : Query[(K, V)]` with a `(k, f)` tuple pattern, but ADR 0120 made bynk nominal ‚Äî no tuple/pair exists. **A:** expose keys through `.entries`/`.keys`/`.values` accessors, **not** two-parameter keyed lambdas ‚Äî `.entries` lifts once into an ordinary `Query` and the whole single-argument vocabulary applies unchanged, the smaller build. **B:** an entry is the nominal `MapEntry[K, V]` record `{ key: K, value: V }` (read `.key`/`.value`), the same "name the shape, no anonymous product" choice ADR 0120 made for join rows. **C:** `MapEntry` is compiler-known (fields resolve like `JsonError`, deferring to a user type of the same name ‚Äî no new reserved word) and **non-boundary** (a generic-record instantiation, ADR 0183), so a read handler projects each entry into a named type before its terminal ‚Äî which is what lets the stored value drop the denormalised key; not annotatable in v1. **D:** the accessors are paren-less builders on the *store-map* field, distinct from the eager in-memory `Map.keys()`/`.values()` (ADR 0035). **E:** a persisted key is a string object-key, so an `Int`-based key decodes with `Number(‚Ä¶)` on read (value-keyable domain, ADR 0110 D5).
